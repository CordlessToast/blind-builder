HOW THE CSS LOADS IN ORDER
==========================

1. variables.css ........... Define all colors and sizes
   ↓
2. animations.css ......... Define animations that other files use
   ↓
3. global.css ............ Body styles, app container
   ↓
4. components/*.css ....... Component specific styles
   ↓
5. pages/*.css ........... Page specific styles
   ↓
6. games/*.css .......... Game screen styles

This order matters! Variables must load first so other files can use them.


FILE NAMING CONVENTION
=====================

JavaScript Files:
  ✓ PascalCase for components: Header.jsx, LobbyPage.jsx
  ✓ camelCase for hooks/utilities: useSocketListeners.js, useGameActions.js
  ✓ camelCase for files in services/utils: socketConfig.js

CSS Files:
  ✓ PascalCase matching component names: Header.css, LobbyPage.css
  ✓ Use simple descriptive names: variables.css, global.css, animations.css
  ✓ Special files: variables.css, global.css, animations.css (always lowercase)

This way, finding styles is easy - component styles match component names!


QUICK REFERENCE
===============

Need to style a new button?
  → Add CSS to: styles/components/Buttons.css
  → Or create new file: styles/components/NewComponent.css

Need to change all primary colors?
  → Edit: styles/variables.css
  → Change: --color-primary: #fbc02d;

Need animation timing?
  → Edit: styles/variables.css
  → Change: --transition-normal: 0.2s;

Need to debug spacing?
  → All spacing values are in: styles/variables.css
  → Look for: --spacing-*

Running into CSS conflicts?
  → Check: styles/global.css
  → Check: styles/components/
  → Check relevant page or game file
